# 秋招项目说明

## 你应该怎么介绍这个项目

这个项目不是普通的自动化测试脚本，而是一个面向 Web 应用缺陷分析的 AI 测试平台。它把用户描述、页面截图、前后端日志、代码片段和操作录屏放在同一个分析流程里，自动输出 Bug 报告、根因分析、可疑代码位置和自动化测试用例。

一句话介绍：

```text
我做了一个多模态 UI 缺陷定位与自动化测试生成平台，可以根据截图、日志、代码和用户描述自动分析 Web 应用 Bug，并生成复现步骤、修复建议和 pytest / Playwright 测试用例。
```

## 面试官可能会问什么

### 1. 为什么说它是多模态？

因为系统不只处理文本，还处理了页面截图、操作录屏、日志、代码和结构化测试结果。截图和录屏提供 UI 状态，日志提供错误证据，代码提供定位依据，用户描述提供业务上下文。

### 2. 当前版本有没有真正调用大模型？

当前版本默认使用规则推理和代码检索，保证没有 API Key 也能完整演示。项目里预留了 `prompts/`，后续可以把截图、录屏关键帧、日志和代码片段交给多模态大模型进一步增强分析。

这个回答反而很稳，因为说明你先做了可运行的工程闭环，再考虑模型增强。

### 3. 怎么定位代码？

系统会解析前后端日志，提取接口路径、错误类型、堆栈文件、函数名和关键词；然后扫描上传代码目录，把代码按函数或块切分，根据关键词和错误类型打分，最后输出最可疑的文件和代码片段。

### 4. 测试用例怎么生成？

系统先识别缺陷类型，例如服务端 500、前端空值异常、接口超时、权限异常、上传限制异常、金额不一致等；然后根据缺陷类型生成对应的后端 `pytest` 和前端 `Playwright` 测试模板。

### 5. 这个项目还有什么可以优化？

可以继续加：

- 接入多模态大模型分析截图和关键帧。
- 使用向量数据库做代码语义检索。
- 支持真实项目 zip 上传。
- 支持 CI 自动运行生成的测试。
- 增加 FastAPI 后端和任务队列。
- 接入浏览器自动复现功能。

## 简历版本

```text
MultiBug-Agent：基于多模态大模型的软件缺陷定位与自动化测试生成平台

设计并实现一个面向 Web 应用测试场景的多模态缺陷分析平台，支持输入页面截图、操作录屏、前后端日志、代码片段与用户描述，自动完成缺陷类型识别、复现步骤生成、根因分析、可疑代码定位，并生成 pytest / Playwright 自动化测试用例和 Markdown Bug 报告。

项目亮点：
1. 支持截图、日志、代码、用户描述和录屏关键帧等多模态输入。
2. 设计日志解析与代码检索流程，提取接口路径、错误类型、堆栈信息和关键词，实现可疑代码定位。
3. 针对服务端 500、接口超时、前端空值异常、权限异常、上传限制、金额不一致等缺陷类型生成测试用例。
4. 基于 Streamlit 构建交互式平台，支持示例场景演示、手动上传、报告导出和测试脚本下载。
```

## 你的演示顺序

1. 打开 Streamlit 页面。
2. 选择 `case_01_register_500`。
3. 展示左侧截图、日志、代码片段。
4. 点击“开始分析”。
5. 解释右侧生成的缺陷类型、证据、复现步骤和可疑代码位置。
6. 展示自动生成的 `pytest` 和 `Playwright` 测试。
7. 再切换到 `case_02_login_timeout`，展示录屏输入和 loading 卡死场景。
